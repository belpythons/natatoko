## 2025-01-20 - Redundant Database Aggregations
**Learning:** While pushing collection mapping to the database via `SUM` queries is a huge performance win for unbound queries, doing so on a dataset that is *already eagerly loaded into memory* for other reasons (e.g. `$sessions->with('consignments')` for returning raw rows to Inertia) is actually a de-optimization. It introduces unnecessary database queries when the data is already hydrated in PHP memory.
**Action:** Before optimizing a collection map loop into a database aggregate query, always check if the collection is loaded and used elsewhere in the same method. If the models must be eagerly loaded regardless, aggregating in PHP memory via collections is faster because it saves an extra database round trip. Only use pure database queries for sums/counts when we *don't* need the underlying models hydrated.
